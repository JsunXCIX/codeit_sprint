## 개요
온라인 쇼핑몰의 `상품 상세 페이지 → 장바구니 추가 → 결제 완료` 흐름을 분석하기 위해 필요한 핵심 이벤트와 속성을 정의함.

<br>

## Event

사용자의 행동을 단계별로 추적하기 위한 이벤트 로그 명세서.

| 분류 | 이벤트 설명 | page | action | event attributes | Trigger |
| :--- | :--- | :--- | :--- | :--- | :--- |
| 상품 | 상품 상세 페이지 조회 | `product_detail` | `view_product` | `product_id` <br> `category` <br> `price` | 상품 상세 페이지 로딩이 완료된 시점 |
| 장바구니 | 장바구니 상품 추가 | `product_detail` | `add_to_cart` | `product_id` <br> `quantity` <br> `referrer` | 장바구니 담기 버튼을 클릭한 시점 |
| 결제 | 최종 주문 완료 | `checkout_success` | `purchase` | `order_id` <br> `total_amount` <br> `payment_method` | 결제 완료 후 결과 페이지가 노출된 시점 |

<br>

## Event Attributes

분석의 정확도와 데이터 처리 효율을 고려하여 각 속성의 데이터 타입을 정의함.

| 속성명 | 설명 | 타입 |
| :--- | :--- | :--- |
| product_id | 상품의 고유 식별 번호 | `int` |
| category | 상품 카테고리 분류 명칭 | `varchar` |
| price | 상품의 단가 | `int` |
| quantity | 장바구니에 담은 수량 | `int` |
| referrer | 이벤트 발생 직전 페이지 경로 | `varchar` |
| order_id | 주문 건의 고유 식별 번호 | `varchar` |
| total_amount | 최종 결제 금액 | `int` |
| payment_method | 선택한 결제 수단 | `varchar` |

<br>

## 나의 인사이트 (*2026년 5월 6일 기준*)
> 현재의 로그 설계 과정은 아직 온전히 내 것으로 체득되지 않은 듯한 모호함이 남아 있다. 지금은 설계 시 "이런 흐름이지 않을까?"라는 상상력에 의존해, 직접 사용자가 된 기분으로 이벤트 속성을 정의하고 있다.

> `product_id`나 `category`처럼 분석의 뼈대가 되는 필수 속성들에 대해서는 확신이 생겼지만, 그 외의 부가적인 속성들을 어느 정도까지 포함해야 할지에 대해서는 여전히 삽입 여부를 결정하기가 어렵게 느껴진다. 있어도 되고 없어도 될 것 같은 데이터들 사이에서 적절한 타협점을 찾는 과정이 아직은 애매한 경험으로 다가온다.

> 하지만 이러한 막연함은 더 정교한 설계를 향한 시작점이라고 생각한다. 앞으로 다양한 비즈니스 상황을 가정해 로그를 설계해보고, 실무적인 피드백을 수용하며 보완해 나가며, 상상에 의존하던 설계를 근거 있는 확신으로 바꿔 나가는 과정을 기록하며 성장할 것이다.
