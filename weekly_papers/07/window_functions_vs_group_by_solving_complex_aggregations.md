# 윈도우 함수(Window Function): 행을 유지하며 집계하기

## 1. Window 함수는 어떤 문제를 해결하기 위해 존재하는가?
전통적인 `GROUP BY` 방식은 데이터를 그룹핑하면 기존의 개별 행 정보가 사라지고 집계된 결과(Summary)만 남게 된다. 

하지만 실무에서는 전체 데이터의 상세 내역을 유지하면서, 동시에 그룹별 통계치를 각 행 옆에 나열해야 하는 경우가 많은 것 같다. Window 함수는 바로 이 상세 내역 유지와 집계라는 두 마리 토끼를 동시에 잡기 위해 존재한다.

---

## 2. Window 함수 vs GROUP BY

| 비교 항목 | GROUP BY | Window Function (OVER) |
| :--- | :--- | :--- |
| **데이터 행 수** | 집계 기준에 따라 줄어듦 | 원본 행 수가 그대로 유지됨 |
| **데이터 디테일** | 개별 행의 상세 정보를 볼 수 없음 | 개별 행 정보와 집계 값을 함께 확인 가능 |
| **주요 용도** | 전체적인 요약 레포트 작성 | 순위 계산, 이동 평균, 누적 합계 등 |

> **작성자의 인사이트:** `GROUP BY`가 데이터를 하나로 뭉쳐서 점수를 매기는 느낌이라면, 윈도우 함수의 `PARTITION BY`는 데이터의 개별성은 살려두되 옆에 참고용 그룹 점수표를 하나씩 붙여주는 느낌이다.

## 3. 실무 활용 시나리오: "카테고리별 매출 Top 3 상품 뽑기"

작성자가 생객해본 시나리오 중 하나인 그룹 내 순위 매기기 예시이다.

### **상황**
피자 가게에서 "각 카테고리별로 가장 많이 팔린 메뉴 3개씩만 뽑아주세요"라는 요청이 들어왔다.

* **GROUP BY를 쓴다면?**
    * 카테고리별 최대 판매량은 알 수 있지만, 그 판매량이 어떤 메뉴의 것인지 상세 정보를 함께 뽑으려면 서브쿼리를 복잡하게 써야 한다.
* **Window 함수를 쓴다면? (`DENSE_RANK` 활용)**
    ```sql
    SELECT 
        category, 
        menu_name, 
        sales_count,
        DENSE_RANK() OVER (PARTITION BY category ORDER BY sales_count DESC) as rank
    FROM pizza_sales;
    ```
    이 쿼리 한 줄이면 각 행 옆에 카테고리 내 순위가 바로 찍힌다. 여기서 `WHERE rank <= 3` 조건만 걸면 아주 쉽게 결과를 얻을 수 있다.

---

## 4. 활용 가능한 또 다른 사례
* **누적 합계:** 날짜순으로 정렬하여 오늘까지의 누적 매출액 계산.
* **이전/다음 값 비교:** 어제 매출과 오늘 매출을 한 행에서 비교하여 증감률 계산.
