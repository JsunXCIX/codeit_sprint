# 📋 나만의 EDA 체크리스트

## 💡 개요: 왜 나만의 체크리스트가 필요한가?

처음에는 결측치나 이상치를 기계적으로 처리하면 끝날 줄 알았습니다. 하지만 준비 없이 분석에 뛰어드니 전처리 단계에서 계속 뒤로 돌아가는 '무한 루프'에 빠지게 되었습니다. 데이터의 전체적인 구조를 먼저 보지 않았기 때문입니다. 시행착오 끝에 정립한, 분석의 흐름을 끊지 않는 5가지 체크리스트를 소개합니다.<br>

## 💡 나만의 체크리스트

### ✅ 데이터 구조 확인 (`shape`, `head()`)

- 전체 행/열의 개수와 상단 데이터를 통해 컬럼의 구성과 데이터의 구조를 파악합니다.
- 내가 다룰 데이터의 크기를 알아야 분석의 방향을 결정할 수 있습니다.

### ✅ 데이터 타입 확인 (`info()`, `dtypes`)

- 각 컬럼의 데이터 타입이 분석 목적에 맞는지 확인합니다.
- 날짜 데이터가 문자열(object)로 되어 있거나, 범주형 데이터가 숫자(int)로 되어 있으면 이후 계산이나 시각화에서 오류가 발생합니다.

### ✅ 매핑 데이터 확인

- 시도 코드(11, 26 등)나 상태값(0, 1)처럼 숫자로 치환된 데이터의 원본 의미를 확인합니다.
- 숫자로 저장된 매핑 데이터는 가독성이 떨어집니다. 분석 초기에 이를 실제 의미(예: 11 → 서울)로 변환하거나 파악해두지 않으면, 분석 내내 메타데이터 문서를 들락날락하며 흐름이 깨지게 됩니다.

### ✅ 범주형 변수와 수치형 변수 확인

- 그룹화의 기준이 될 변수(범주형)와 연산의 대상이 될 변수(수치형)를 분류합니다.
- 이 과정을 거치면 자연스럽게 'A별 B의 추이' 같은 분석 시나리오가 머릿속에 그려지고, 전처리를 하면서 동시에 분석의 가설을 세울 수 있게 됩니다.

### ✅ 결측값 확인 (`isna().sum()`)

- 컬럼별 결측치 개수와 전체 대비 비중을 파악합니다.
- 결측치는 삭제와 대치 중 선택이 필요합니다. 비중을 먼저 파악해야 "이 컬럼은 살릴 것인가, 과감히 포기할 것인가"에 대한 판단이 가능해집니다.<br>

## 🧠 EDA에 관한 개인 경험과 느낀점
실제로 결측치와 이상치 처리 사이에서 고민하며 깨달은 점이 있습니다. 모든 컬럼의 결측치를 완벽히 처리하려다 보면 정작 중요한 데이터까지 함께 삭제되는 경우가 많았습니다.
그래서 저는 주제별 데이터 처리를 하는 것이 좋다고 생각합니다. 분석 목적에 맞는 컬럼들만 먼저 추출하여 새로운 데이터 프레임을 만들고, 그 범위 안에서 집중적으로 전처리를 진행하는 것이 데이터 손실을 최소화하고 분석 효율을 극대화할 수 있다고 느꼈습니다.

# 📊 Tableau: 사용자와 대화하는 인터랙티브 시각화

## 🏠 상상 시나리오: 회의 중 발생한 상사의 돌발 질문
> ### 상사: "2025년 매출이 왜 이렇게 저조했지? 지금 바로 월 단위로 쪼개서 볼 수 있나?"
만약 정적인 이미지나 PPT 차트로만 준비했다면, 아마 "데이터 다시 확인해보고 따로 보고드리겠습니다"라고 말하며 일이 늘어날 것이다. 하지만 Tableau의 인터랙티브 기능이 있다면 그 자리에서 바로 클릭 몇 번으로 답변이 가능해집니다.<br>

## 활용할 인터랙티브 기능과 선정 이유
### 날짜 계층 구조 변경
* Tableau의 열에 올린 날짜 데이터 옆의 `+` 버튼을 클릭하여 분기 → 월 → 일 단위로 즉시 확장.
* "매출이 왜 낮아졌지?"라는 질문에 대해, 특정 시점의 문제를 파악하기 위해 차트를 새로 만들 필요 없이 클릭 한 번으로 분석이 가능하기 때문
